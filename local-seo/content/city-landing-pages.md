# City Landing Pages - Content Briefs

## Strategy

Each city page serves as a local SEO entry point targeting "[service] + [city]" searches. These pages must feel locally relevant, not like a template with a swapped city name. Every page pushes toward one CTA: book a call via Calendly.

URL structure: `/[city]/`
Schema: LocalBusiness + Service
CTA: "Reservez un appel decouverte" (linked to Calendly)

## Cross-Page Rules

1. **No duplicate content.** Each page must have unique H1, unique intro, unique local context. Shared sections (services list, process) can be structurally similar but must be rewritten per city.
2. **hreflang not needed.** All pages are in French targeting Belgium/Luxembourg.
3. **Internal linking.** Each city page links to all service pages. City pages link to each other in a "Nous servons aussi" footer section.
4. **Schema.** Every city page gets LocalBusiness schema pointing to P-XEL's Liege address, with `areaServed` set to the target city.
5. **CTA consistency.** Every page uses the same Calendly link. CTA text can vary but action is always "book a discovery call."
6. **Mobile first.** These pages will be entry points from Google Maps and local searches, often on mobile. Design accordingly.
7. **Page speed.** No heavy images above the fold. Portfolio screenshots lazy-loaded. Target < 2s LCP.
